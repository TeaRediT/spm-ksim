## Внесені зміни у вихідну логіку моделі, за варіантом:

1. Додано можливість вказання початкової кількості хмар. В інтерфейс додано слайдер initial-clouds (діапазон 0-10). У процедуру setup внесено зміни, щоб при запуску моделі створювалася відповідна кількість хмар:

```nlogo
to setup
  clear-all
  setup-world
  set temperature 12
  ;; Створення початкової кількості хмар згідно зі слайдером
  repeat initial-clouds [ add-cloud ]
  reset-ticks
end
```

2. Реалізовано зміну кута падіння сонячних променів. Для імітації руху сонця протягом доби додано логіку зміни кута падіння променів залежно від модельного часу (ticks). Цикл розділено на три фази: ранок, день та вечір. Змінено процедуру create-sunshine:

```nlogo
to create-sunshine
  let sun-angle 160 ;; Стандартний кут

  ;; логіка зміни кута (цикл 300 тактів)
  let local-time ticks mod 300
  if local-time < 100 [ set sun-angle 120 ]      ;; Ранок
  if local-time >= 100 and local-time < 200 [ set sun-angle 180 ] ;; День
  if local-time >= 200 [ set sun-angle 240 ]     ;; Вечір

  if 10 * sun-brightness > random 50 [
    create-rays 1 [
      set heading sun-angle
      set color yellow
      setxy (random 10) + min-pxcor max-pycor
    ]
  ]
end
```

## Внесені зміни у вихідну логіку моделі, на власний розсуд:

Механіка "Перемикання Альбедо" (Танення льодовиків). Додано логіку, що імітує танення льодовиків при досягненні високої температури. Якщо температура перевищує поріг, альбедо різко падає, що відображає перетворення льоду на воду. До процедури go додано виклик:

```nlogo
to update-dynamic-albedo
  ;; Якщо температура > 35, альбедо падає до 0.3 (лід розтанув)
  if temperature > 35 [ set albedo 0.3 ]

  ;; Якщо температура впаде < 15, альбедо повертається до 0.6 (лід намерз)
  if temperature < 15 [ set albedo 0.6 ]
end
```
